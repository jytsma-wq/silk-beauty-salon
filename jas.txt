### 📌 PROMPT 8: API Architecture & Documentation

```
Task: Refactor and document API architecture with versioning, validation, and OpenAPI specification

Context:
- Current API routes in /src/app/api/ lack structure
- No API documentation
- Missing validation middleware
- No versioning strategy

Requirements:

1. **API Structure Refactoring**
   - Organize API routes by version:
     * /api/v1/contact
     * /api/v1/booking
     * /api/v1/newsletter
     * /api/v1/treatments
   - Create shared middleware:
     * Request validation
     * Authentication (if needed)
     * Rate limiting
     * CORS handling
     * Response formatting
   - Implement consistent error responses
   - Add request/response logging

2. **Request Validation**
   - Use Zod for all API route validation
   - Create schemas in /src/schemas/api/
   - Validate:
     * Request body
     * Query parameters
     * Headers
     * Path parameters
   - Return detailed validation errors
   - Add validation middleware

3. **Response Standardization**
   - Create standard response wrapper:
     ```typescript
     {
       success: boolean;
       data?: any;
       error?: {
         code: string;
         message: string;
         details?: any;
       };
       meta?: {
         requestId: string;
         timestamp: string;
         version: string;
       };
     }
     ```
   - Implement in all API routes
   - Add TypeScript types for responses

4. **OpenAPI/Swagger Documentation**
   - Generate OpenAPI 3.0 specification
   - Use swagger-jsdoc or similar
   - Document all endpoints:
     * Description
     * Parameters
     * Request body schema
     * Response schema
     * Error responses
     * Examples
   - Add interactive API documentation UI (Swagger UI)
   - Auto-generate TypeScript types from spec

5. **API Versioning Strategy**
   - Implement version in URL path
   - Add deprecation headers
   - Maintain backward compatibility
   - Document breaking changes
   - Create migration guides

6. **Rate Limiting Per Endpoint**
   - Different limits for different endpoints:
     * Form submissions: 5/15min
     * Data fetching: 100/min
     * Authentication: 10/min
   - Per-user rate limiting
   - Add rate limit headers in response

7. **API Testing**
   - Create Postman/Insomnia collection
   - Add integration tests for all endpoints
   - Add contract testing
   - Test error scenarios
   - Load testing for performance

8. **API Security**
   - Implement API key authentication (if needed)
   - Add CORS configuration
   - Input sanitization
   - Output encoding
   - SQL injection prevention
   - Rate limiting

9. **Webhook Support** (if applicable)
   - Implement webhook endpoints
   - Add webhook signature verification
   - Retry logic for failed webhooks
   - Webhook event logging

Deliverables:
- Refactored API structure in /src/app/api/v1/
- Validation schemas in /src/schemas/api/
- Middleware in /src/lib/api/middleware/
- OpenAPI specification in /openapi.yaml
- API documentation UI at /api-docs
- Postman collection
- API testing suite
- API client SDK (optional)
- Documentation in docs/API.md

API Standards:
- RESTful conventions
- JSON request/response
- UTF-8 encoding
- ISO 8601 dates
- Pagination for lists
- Proper HTTP status codes
- HATEOAS where appropriate
```
