### 📌 PROMPT 5: Advanced State Management & Data Fetching

```
Task: Implement robust state management and optimized data fetching strategy

Context:
- Currently using basic React context and props drilling
- Need global state management for complex features
- Optimize data fetching with caching and deduplication

Requirements:

1. **State Management Implementation**
   - Integrate Zustand for global state (lightweight alternative to Redux)
   - Create stores for:
     * User preferences (language, theme, consent)
     * Booking flow state
     * Form state management
     * UI state (modals, drawers, notifications)
     * Shopping cart (if applicable)
   - Implement persistence with zustand/middleware
   - Add DevTools integration for debugging
   - Type-safe store definitions

2. **Data Fetching Strategy**
   - Integrate TanStack Query (React Query) for server state
   - Implement data caching strategy
   - Add optimistic updates for better UX
   - Implement background refetching
   - Add stale-while-revalidate pattern
   - Configure cache invalidation strategies
   - Add offline support with persistence

3. **Form State Management**
   - Integrate React Hook Form for complex forms
   - Add Zod schema validation
   - Implement multi-step forms for booking
   - Add form persistence (local storage)
   - Implement field-level validation
   - Add async validation (email/phone check)
   - Create reusable form components

4. **Real-time Updates** (if needed)
   - Add WebSocket support for live bookings
   - Implement Server-Sent Events for notifications
   - Add optimistic UI updates
   - Sync state across tabs

5. **Performance Optimization**
   - Implement memoization strategies
   - Add selector optimization
   - Use shallow comparison for state updates
   - Implement virtualization for large lists
   - Add pagination/infinite scroll

6. **Developer Experience**
   - Add state debugging tools
   - Create custom hooks for common patterns
   - Add TypeScript types for all state
   - Document state flow diagrams
   - Add state migration utilities

Deliverables:
- Zustand stores in /src/stores/
- React Query configuration in /src/lib/react-query.ts
- Form schemas in /src/schemas/
- Custom hooks in /src/hooks/
- Form components in /src/components/forms/
- State persistence utilities
- WebSocket client (if applicable)
- DevTools configuration
- Documentation in docs/STATE_MANAGEMENT.md

State Architecture:
- Server state: React Query (API data, database queries)
- Global client state: Zustand (user preferences, UI state)
- Local component state: useState/useReducer
- Form state: React Hook Form
- URL state: Next.js router (search params, query strings)
```
