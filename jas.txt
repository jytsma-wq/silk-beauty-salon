### 📌 PROMPT 6: Enhanced Accessibility (WCAG 2.1 AA Compliance)

```
Task: Achieve full WCAG 2.1 Level AA accessibility compliance

Context:
- Current implementation has basic accessibility
- Need comprehensive accessibility for all users
- Support for screen readers, keyboard navigation, and assistive technologies

Requirements:

1. **Keyboard Navigation**
   - Ensure all interactive elements are keyboard accessible
   - Implement proper focus management
   - Add visible focus indicators
   - Create skip links for main content
   - Implement roving tabindex for complex widgets
   - Add keyboard shortcuts documentation
   - Test with keyboard-only navigation

2. **Screen Reader Support**
   - Add comprehensive ARIA labels
   - Implement ARIA live regions for dynamic content
   - Add ARIA descriptions where needed
   - Test with NVDA, JAWS, and VoiceOver
   - Add screen reader-only text where helpful
   - Ensure proper heading hierarchy
   - Add landmark regions

3. **Color Contrast**
   - Ensure WCAG AA contrast ratios (4.5:1 for text, 3:1 for UI)
   - Add high contrast mode
   - Test all color combinations
   - Provide alternative visual indicators (not just color)
   - Test with color blindness simulators

4. **Forms Accessibility**
   - Add proper label associations
   - Implement error message announcements
   - Add fieldset and legend for grouped inputs
   - Provide inline validation feedback
   - Add required field indicators
   - Implement autocomplete attributes
   - Add input format hints

5. **Media Accessibility**
   - Add captions for videos
   - Provide text alternatives for images
   - Add transcripts for audio content
   - Implement controls for autoplay media
   - Add audio descriptions where needed

6. **Responsive & Zoom Support**
   - Support 200% zoom without loss of functionality
   - Ensure text spacing adjustments work
   - Test with different viewport sizes
   - Support orientation changes
   - Avoid horizontal scrolling

7. **Motion & Animations**
   - Respect prefers-reduced-motion
   - Provide pause controls for animations
   - Avoid seizure-inducing content
   - Add alternative static content

8. **Automated Testing**
   - Integrate axe-core for automated testing
   - Add pa11y for CI/CD pipeline
   - Implement Lighthouse accessibility audits
   - Add accessibility tests to E2E suite
   - Create accessibility test reports

9. **Documentation**
   - Create accessibility statement page
   - Document keyboard shortcuts
   - Add accessibility guidelines for contributors
   - Provide feedback mechanism for accessibility issues

Deliverables:
- Accessibility audit report
- Enhanced components with ARIA in /src/components/
- Accessibility utility functions in /src/lib/a11y.ts
- Skip link component (enhanced)
- Focus trap utility
- Accessibility testing suite
- High contrast theme
- Accessibility statement page at /[locale]/accessibility
- Keyboard shortcuts documentation
- Screen reader testing report
- Documentation in docs/ACCESSIBILITY.md

Testing Checklist:
- ✓ Keyboard navigation
- ✓ Screen reader testing (NVDA, JAWS, VoiceOver)
- ✓ Color contrast validation
- ✓ Automated axe-core audits
- ✓ Mobile accessibility
- ✓ Form accessibility
- ✓ Focus management
- ✓ ARIA implementation
```
